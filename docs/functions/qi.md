# Qi 用户手册

> 适用版本：Mantrika Tools（当前主线）

---

## 1. 概述

**Qi** 是一个**素材变体生成器**——选中工程里若干 item，按下 Generate，它会按你设置的随机/递进规则，沿时间线**复制出 N 份风格各异的变体**：每份的音量、音高、声像、是否反向、起点偏移、甚至挂载的 FX 参数都不同。

它适合的场景：

- 一段脚步声、刀剑碰撞、爆炸，想要 **20 份永不重复**的变体铺到时间线上
- 一段连续的素材，想要 **顺次过渡**从 -3 半音到 +3 半音的渐进感
- 一组 Foley 素材，想要每份都从原始素材里**随机截取一小段**而不是整段播放
- 想要**先听几遍变化**再决定要不要保留——Qi 在你按 Generate 前提供完整可试听的蓝色预览
- 这一套"配方"经常用，**存成预设**下次一键调出来

整个窗口的核心工作流：**选 item → 调参数 → 听预览 → Generate 落地**。

---

## 2. 打开方式

菜单入口：

```
Extensions → MantrikaTools → Qi
```

或在 Action List 搜：

| Action 名称 | 用途 |
| --- | --- |
| **`mantrika : Misc - Qi`** | 打开 / 关闭 Qi 窗口 |
| **`mantrika : Qi: Apply Preset 1`** ~ **`5`** | 不开窗，直接对选中 item 应用 1~5 号预设 |

> 💡 五个 Preset Action 可以绑定快捷键——选中 item → 按快捷键 → 直接出结果，不进窗口。

## 4. 选 item 与预览

### 4.1 选中 Item

在 REAPER 主时间线上选中一个或多个 item，然后切到 Qi 窗口——Qi 把这些 item 作为**源素材**，所有变体都基于它们生成。

> 💡 选中**多个 item** 时，Qi 会把它们当作"**一组**"复制：每份变体都包含这一整组里的全部 item，组内 item 的相对位置和长度被保留。

### 4.2 蓝色预览（Preview）

只要拖动了任意参数滑条，Qi 就在时间线上**自动生成一组蓝色 item**——这是预览，还没真正提交。蓝色的含义是"临时"，可以随便改。

- 改完参数：蓝色 item **自动重新计算**
- 觉得满意：按下 **GENERATE**，蓝色变成正常颜色，变成永久 item
- 不想要了：按 **Reset** 按钮（右上角），蓝色全部消失，参数回到默认，或者直接把Generate拉到0。

> ⚠️ **撤销 (Ctrl+Z) 与 Qi 的预览**——Qi 会自动清理 undo History中的预览记录，不会留下旧蓝色 item 污染工程。

### 4.3 试听三键

Footer 左侧的 `<` `R` `>` 三个小按钮用来切换播放哪一份变体：

| 按钮 | 行为 |
|---|---|
| `<` | 切到上一份变体并播放 |
| `R` | 随机跳到任意一份并播放 |
| `>` | 切到下一份变体并播放 |

每次切换都会**Solo 那一份**，方便你 A/B 对比。空格键也能暂停/恢复当前预览。

## 5. Logic Bar（顶部行）

### 5.1 MODE：随机 vs 递进

```
[ Ran | Seq ]
```

- **Ran（Random）**——每份变体的参数都是**真随机**，互相之间没有关系
- **Seq（Sequential）**——参数从 min 线性走到 max，呈**渐进**变化

切到 **Seq** 时会冒出第二个开关：

```
[ Lo-Hi | Hi-Lo ]
```

控制递进方向：从低值走到高值，还是从高值走到低值。

> 💡 用 Seq 做"渐强的脚步声"、"Pitch逐渐上升的蓄力等"非常方便。

### 5.2 GLOBAL ENV：总 Envelope 开关

```
[ ENV ]
```

控制是否**为变体内部画 Take Envelope 包络**。关闭时，每份变体的 Vol/Pitch/Pan 在变体长度内**保持一个固定随机值**；打开时，会在变体内部画出多点 envelope，让数值**在变体内部动态起伏**。

各 Strip 上的 Env 副滑条（点数）需要这个总开关打开才生效。

### 5.3 Preset 按钮

打开预设面板，详见 §10。

### 5.4 Reset 按钮

把所有参数恢复到默认、清掉蓝色预览。

## 6. Strips（六条横条）

每条 Strip 都是同样的结构：

```
[●指示灯]  STRIP名  [主滑条]  数值  [Env副滑条]  Env数值  [▶展开]
```

- **左侧圆点指示灯**——点一下**启用/禁用**该 Strip，灰色 = 禁用
- **主滑条**——该参数的随机幅度/强度
- **Env 副滑条**——envelope 点数（多少个起伏点）
- **▶ 展开箭头**（如有）——展开后会露出更多细节设置

> 💡 **滑条修饰键**（适用于所有参数滑条，包括 Footer 的 VARIANTS / SPACING）：
> - **Shift + 拖动** = 精调，拖动幅度变细，方便摸细节
> - **Alt + 拖动**（仅 PAN 双值滑条） = 整体平移，左右把手一起动，保留区间宽度
> - **Ctrl + 左键 数字**（不是滑条，是滑条旁边那串数值文字）= 数字**就地变成输入框**，敲新数字 + Enter 提交。范围超界自动 clamp 到物理上下限；Esc / 点其它地方 = 取消
>   - 编辑器打开时整段文字已选中，直接敲新值就替换
>   - PAN 双值数字用 `L50 ~ R50` / `C ~ R30` / `-0.5 ~ 0.5` 三种语法都行，中间用 `~` 隔开。任何一段解析失败整体放弃
>   - env 副数字输入 `0` 等同 OFF

### 6.1 VOL：音量随机

主滑条范围 **0~12 dB**。值 `±X dB` 表示每份变体的音量在 **-X 到 +X dB** 之间随机/递进。

副滑条：envelope 点数（0~10）。打开 **GLOBAL ENV** 后，每份变体内部会画出 N 个音量包络点，做"变体内动态忽大忽小"。

### 6.2 PIT：音高随机

主滑条范围 **0~24 半音**。值 `±X st` 表示音高在 **-X 到 +X 半音**之间变化。

**展开内容（▶）**：

```
[ Rnd | Custom | Keep ]    [Pitch Algorithm ▼]    [M]
```

- **Rnd**：每份变体使用**随机算法**（拉伸时音质多样化）
- **Custom**：使用下拉框里选定的固定算法（Elastique Pro 等）
- **Keep**：保留每个源 item 原本设置的算法
- **[M] 按钮（STR/PIT 切换）**：打开后 Strip 标题从 `PIT` 变 `STR`——切换到 **Stretch 模式**：变体长度变化（实际是变速），原本"变调不变速"变成"变速变调"

> ⚠️ Stretch 模式下，音高 Envelope 会切换成拉伸的包络（变速和包络拉伸不兼容）。

### 6.3 PAN：声像随机

主滑条是**双把手**滑条，可分别设左右边界：

- `L50 ~ R50` 表示左 50% 到右 50% 范围内随机
- `C ~ R80` 表示中央到右 80% 范围内随机
- 双击滑条把手可重置

副滑条：envelope 点数，让变体内 Pan 来回飘移。

### 6.4 MOD：反向 + 起点偏移

```
[ Off | Rnd | Next ]    Rev 50% | Ofs 100ms
```

**头部分段开关（Segment 模式）**——控制**变体使用源素材的哪一段**：

| 模式 | 行为 |
|---|---|
| **Off** | 每份变体使用源素材的**完整一段**（默认） |
| **Rnd** | 每份变体**随机截取**源素材的一小段（按 SPACING 长度切片） |
| **Next** | 每份变体**顺次取**源素材的下一段（适合长素材做"分块铺设"） |

> 💡 用 Rnd / Next 做长 Foley 素材的"切碎重新铺设"特别好用——不用手动剪。

**展开内容（▶）**：

- **Reverse 滑条（0~100%）**——每份变体**反向播放的概率**。100% 全部反向，50% 一半的变体反向。
- **Offset 滑条（0~1000 ms）**——每份变体的**起点偏移随机量**。比如设 200ms，每份变体的起点会在 ±200ms 内随机晃动，做出"节奏不齐"的人化效果。

> 💡 整条 MOD Strip 的指示灯（点左边圆点）控制的是 Reverse 是否启用；Offset 在展开区独立。

### 6.5 FX：FX 参数随机

Strip 头显示 `N params`——你已经"捕获"了 N 个 FX 参数让 Qi 调控。

**展开内容（▶）**：

```
Pts: ━━━ 3 pts    Depth: ━━━ 30 %
[ N params ]  [Listening...]  [+ CAPTURE] [OK]
─────────────────────────────────────────
[Item 1] ReaEQ - Band1 Freq         ×
[Item 1] ReaComp - Threshold        ×
...
```

- **Pts 滑条**——FX envelope 在变体内画几个起伏点
- **Depth 滑条（0~100%）**——FX envelope 的**摆动幅度**（占参数全行程的百分比）
- **+ CAPTURE 按钮**——按下后进入"监听模式"（按钮变成 **OK**），此时**你去 REAPER 里任意拨动一个 FX 参数**，Qi 会把它**自动捕获**到这个列表里；按 **OK** 退出监听
- **× 图标 / 双击 / Delete 键**——从列表里移除一个 FX 参数

> 💡 **使用流程**：选中源 item → 给它挂上想随机的 FX → 在 Qi 里点 CAPTURE → 在 REAPER 里轻轻拨动 FX 参数 → Qi 自动记录 → 重复 → 点 OK 完成。之后每份变体都会在你捕获的 FX 参数上随机摆动。

### 6.6 AMT（Amount）：总幅度宏

```
●  AMT   ━━━━━━━━━━━━━━━ 100%    ━━━━━━━━━ 100%
                          主幅度宏           env密度宏
```

最底下的一条 Strip。两个滑条都是**0~200%** 的**全局缩放器**：

| 滑条 | 影响 |
|---|---|
| **主滑条**（左） | 同时缩放 VOL/PIT/PAN/Offset/FX Depth 的随机幅度 |
| **副滑条**（右） | 同时缩放 VOL/PIT/PAN/FX 的 envelope 点数 |

100% = 原样；0% = 完全去随机化（所有变体一模一样）；200% = 双倍幅度（被各项物理上限 clamp）。

**指示灯点一下** = bypass，整条 AMT 不生效。

> 💡 **典型用法**：把 VOL/PIT/PAN/Offset 都设到你期望的"最大效果"，然后用 AMT 主滑条**一手控制总变化量**——从"很微妙"到"完全失控"无级调节。

## 7. Footer（底部）

### 7.1 VARIANTS：要几份变体

```
VARIANTS
  20   ━━━━━━━━━━━
```

滑条 **0~50**，非线性分段：

| 段位 | 滑条行程 | 用途 |
|---|---|---|
| 0~3 | 前 25% | 细调，做少量精挑细选 |
| 3~10 | 25%~50% | 日常工作量 |
| 10~20 | 50%~75% | 批量铺设 |
| 20~50 | 75%~100% | 大量产出 |

低段细调、高段粗调，符合实际使用习惯。

### 7.2 SPACING：变体间距

```
SPACING [START | GAP]
  0.50s   ━━━━━━━━━━━
```

控制每份变体之间的**时间距离**，单位秒。

滑条 **0~10s**，和 VARIANTS 一样**非线性分段**——小间距是日常工作区，给更多行程细调：

| 段位 | 滑条行程 | 用途 |
|---|---|---|
| 0~0.5s | 前 25% | 极小间距，密集 / 重叠铺设 |
| 0.5~2s | 25%~50% | 日常常用间距 |
| 2~5s | 50%~75% | 拉开的间距 |
| 5~10s | 75%~100% | 大间距，粗调 |

低段细调、高段粗调，和 VARIANTS 思路一致。

`START / GAP` 按钮切换间距的两种含义：

| 模式 | 含义 |
|---|---|
| **START** | 每份变体**起点**之间的间隔（可能重叠） |
| **GAP** | 每份变体**结束**到下一份起点的间隔（永不重叠） |

> 💡 做密集铺设用 START（允许重叠出"踩踏感"）；做规整序列用 GAP。

### 7.3 试听 + GENERATE

```
[<] [R] [>]    [        GENERATE        ]
```

- **`<` `R` `>`**：上一份/随机/下一份（详见 §4.3）
- **GENERATE 按钮**：把当前蓝色预览**提交**为正式 item，参数清零回到默认
- **Enter 键**：等同 GENERATE

> ⚠️ Generate 后**所有参数和源 item 锁定都会清零**——这是设计意图，"提交完一轮就开始新一轮"。

## 8. Mask Tab：按 item 屏蔽参数

切到 **Mask** Tab 看到一张表格：

```
Name              Vol  Pitch  Pan  Rev  Ofs  | FXparam1  FXparam2 ...
──────────────────────────────────────────────────────────────────────
Footstep_01.wav   ☑    ☑      ☑    ☑    ☑   |   ☑          ☑
Footstep_02.wav   ☑    ☐      ☑    ☐    ☑   |   ☑          ☐
...
```

每行对应一个源 item，每列对应一种参数。**取消勾选 = 这个 item 不参与那项随机**。

**典型用法**：

- 一组素材里有几条**已经定好音高**，不想让 Qi 再去随机它们的 Pitch → 取消那几行的 Pitch 勾选
- 某条素材**反向后听起来很差** → 取消它的 Rev 勾选

> 💡 FX 列只在你 CAPTURE 了 FX 参数后才会出现，列名是参数名截断到 6 字符；鼠标悬停可看完整名。

## 9. Browser Tab：素材库浏览

切到 **Browser** Tab，能搜索并拖拽你预先配置的素材库。

```
Database: [ 3 / 5 selected ▼ ]    [ Random ]    [Off|Seg|All]   [Chaos]
Search:   [______________________________]   (124)
──────────────────────────────────────────────
  ● Footstep_Wood_01.wav
  ● Footstep_Wood_02.wav    ← 已使用过的素材会有橙点
    Footstep_Stone_03.wav
    ...
──────────────────────────────────────────────
[                波形视图                  ]
```

### 9.1 顶栏

| 控件 | 用途 |
|---|---|
| **Database 按钮** | 选择启用哪几个素材库（来自 REAPER 自带 Media Explorer 的数据库列表） |
| **Random** | 在搜索结果里随机挑一条预览 |
| **Off / Seg / All** | 拖入时是否应用 Segment 切片：Off=不切；Seg=切一段；All=切全部段 |
| **Chaos** | 拖入时**自动应用 Qi 的当前参数随机化**——拖出去就已经是变体了 |
| **Search** | 实时搜索关键字过滤 |

### 9.2 文件列表

- **单击**：试听该文件（波形区开始播放）
- **拖拽**：拖到 REAPER 轨道上释放，可以拖多个（>15 个会拉一个弹框给你确认）
- **橙色小点**：标记该文件**这一次会话里已经被拖过**了，避免重复使用

### 9.3 波形视图

底部的小波形条用于**精细试听 + 截取片段拖入**：

| 操作 | 行为 |
|---|---|
| **右键单击** | 跳转播放头到此处 |
| **左键拖动** | 框选一段区域（橙色高亮 + 时长标签） |
| **左键拖动选区到 REAPER** | 把**选区那一段**插入轨道（不是整个文件） |
| **空格键** | 暂停/恢复试听 |

> 💡 用这个功能做"我只要这个长素材里 0.5 秒的爆破段"——框选 → 拖到轨道 → 完成。

## 10. Preset 系统

点 Logic Bar 右上角的 **Preset** 按钮：

```
┌─ Qi Presets ───────────────────────────────┐
│  Slot 1  ★ Quick Shotgun        [★] [S] [R] [X] │
│  Slot 2    Heavy Reverse Pack    [★] [S] [R] [X] │
│  Slot 3    (empty)                  [Save here]   │
│  Slot 4    (empty)                  [Save here]   │
│  Slot 5    (empty)                  [Save here]   │
└────────────────────────────────────────────┘
```

**五个固定槽位**。每个槽位：

| 按钮 | 行为 |
|---|---|
| **行体单击**（已存槽位） | 加载该预设 |
| **★** | 设为默认预设（金色 = 当前默认）。打开窗口时自动加载 |
| **S** | 保存当前参数到这个槽（已存的会覆盖） |
| **R** | 重命名 |
| **✕** | 删除 |

**Save here 按钮**（空槽位）：保存当前参数到这个空槽。

### 10.1 默认预设的用途

设了默认预设后，每次打开 Qi 窗口都会自动加载这套参数——做声音类型相对固定的工作时，省去每次手动调参数。

### 10.2 Action 直接应用

```
mantrika : Qi: Apply Preset 1   (~~ 5)
```

这 5 个 Action **不开窗**——选中 item、按快捷键 → 直接生成。适合"我已经知道这套预设效果，看也不用看"的场景。

## 11. 典型工作流

### 工作流 A：50 份永不重复的脚步声

```
1. 选中一段脚步声 item
2. 打开 Qi
3. VOL 拖到 ±3 dB
4. PIT 拖到 ±2 st
5. MOD 头部点 "Rnd"（随机切片）
6. MOD 展开，Reverse 拖到 10%（10% 概率反向）
7. VARIANTS 拖到 50
8. SPACING 拖到 0.4s
9. 按 < > R 听几次满意
10. 按 GENERATE（或回车）
```

---

### 工作流 B：渐进的扫弦

```
1. 选中一个吉他扫弦 item
```

## 12. 注意事项

### 12.1 蓝色预览不是最终结果

蓝色 item = 临时预览。**只有按下 GENERATE 才会变成正式 item**。直接关闭 Qi 窗口或者按 Reset，蓝色会全部消失。

### 12.2 Generate 后所有参数会清零

这是设计意图——一次提交 = 一轮结束。如果想保留参数，先**存成 Preset**。

### 12.3 多选 item 时是"作为一组"复制

不是把每个 item 单独变体，而是把"这一组"作为一个单位整体复制 N 份。组内 item 的相对位置和长度被保留。

### 12.4 Stretch 模式会改变变体长度

打开 Pitch Strip 的 **[M]** 按钮 → Strip 标题变 STR → **变速变调**（拖一根 Stretch Marker），变体长度会因为 PIT 随机值而变化。同时 Pitch Envelope 自动失效。

### 12.5 FX 参数捕获前要先挂 FX

CAPTURE 监听只能捕获**当前选中 item 上已存在的 FX**。如果监听时拨动空气，什么都不会被记录。

### 12.6 AMT = 0% 时所有变体一模一样

不是 Qi 坏了——0% 表示"零随机化"，每份变体都用 base 值。要恢复随机性，把 AMT 主滑条拖回 100%。

### 12.7 Mask 列里 Vol/Pitch/Pan/Rev 的全局开关也会影响

如果某个 Strip 的指示灯（圆点）被你点灭了，那么 Mask 表格里对应的列也会变成**禁用**（灰色不可点）——因为参数本身关了，按 item 屏蔽就没意义。

## 13. 故障排查

| 现象 | 可能原因 | 解决 |
|---|---|---|
| 选了 item 但没出蓝色预览 | VARIANTS = 0 | 把 VARIANTS 拖到至少 1 |
| 蓝色预览不更新 | 参数没动 / Qi 还在 debounce | 等 0.1 秒；或拖一下滑条触发更新 |
| GENERATE 没反应 | 没有蓝色预览 | 先调整参数让蓝色预览出现 |
| 听不出变化 | AMT 主滑条 = 0% / 所有 Strip 都被禁用 | 检查 AMT、检查各 Strip 指示灯 |
| 试听三键灰着 | 还没有蓝色预览 | 调参数让预览出现 |
| FX CAPTURE 不工作 | 当前选中 item 没挂 FX，或者拨的是别的 item 的 FX | 确认你拨的是源 item 上的 FX |
| Stretch 模式下没听到 Pitch Envelope | 设计如此 | Stretch 与音高包络不兼容，用 PIT 主滑条即可 |
| Preset Action 报 "Preset is empty" | 那个槽位还没存东西 | 先在窗口里存进去 |
| Preset Action 报 "Please select items first" | 没选 item | 先选中至少一个 item 再按快捷键 |
| Browser 列表空白 | 数据库没启用，或者关键字过严 | 点 Database 按钮检查启用项；清空 Search |
| 拖入文件后变体没随机化 | Chaos 没勾 | 勾上 Chaos 再拖 |
| 多份变体重叠在一起 | SPACING 是 START 模式且数值过小 | 改成 GAP 模式，或者拉大 SPACING 数值 |
| 撤销后蓝色 item 复活 | undo 还原了一个 generate 操作 | Qi 会自动清掉它们，但要等一拍 |
